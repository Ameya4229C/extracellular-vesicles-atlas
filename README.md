# Interactive figures · *The status of extracellular vesicles as drug carriers and therapeutics*

Companion site for Chaudhari et al., **Nature Reviews Bioengineering** (2026).
DOI: [10.1038/s44222-026-00405-x](https://doi.org/10.1038/s44222-026-00405-x).

The site replots Figures 1–5 from the review as interactive Plotly charts.
Hover any point for details. On the per-study figures (clinical trials, loading-capacity
studies, siRNA dose-response studies) **clicking a point opens the underlying paper or
trial registration** in a new tab.

## What's in here

| Path                | What it is                                                                 |
|---------------------|-----------------------------------------------------------------------------|
| `index.html`        | Single landing page with all three figures and the article header           |
| `css/style.css`     | Site styling                                                                |
| `js/plots.js`       | All Plotly rendering and click handlers                                     |
| `js/fig*.js`        | Inlined data — one per (sub)figure, generated by `build_data.py`            |
| `data/*.csv`        | Original source CSVs (Figs 1–3) plus Fig 4/5 CSVs exported from the xlsx    |

Aggregate bars (article counts in Figs 1a, 1b, 3a) are **hover-only** — the underlying
papers per bar aren't in the CSVs, so there's nothing to link to point-by-point.

## Running locally

The site is fully static and inlines its data into JS, so no server is required:

```sh
open index.html   # double-click also works on macOS
```

For a friendlier dev loop:

```sh
python3 -m http.server 8765 --directory .
# then open http://localhost:8765
```

## Updating the figures

1. Edit the CSVs in `data/`.
2. Re-run the preprocessor from the project root (one level up from this folder):

   ```sh
   python3 build_data.py
   ```
   That regenerates `js/fig*.js`.
3. Reload the page.

## Deploying to GitHub Pages

1. Initialize a repo and push:

   ```sh
   git init && git add . && git commit -m "Initial commit: interactive EV figures"
   git branch -M main
   git remote add origin git@github.com:<you>/<repo>.git
   git push -u origin main
   ```

2. In **Settings → Pages**, set the source to *Deploy from a branch*, branch `main`,
   folder `/ (root)`. After ~1 minute the site is live at
   `https://<you>.github.io/<repo>/`.

3. Once your paper's journal URL is finalised, edit
   [`index.html`](index.html) and replace the `#` in
   `<a id="paper-link" href="#" ...>` with the journal URL. Also remove the
   `<span class="todo">…</span>` placeholder note.

## Notes on the data

- **Fig 1a / 1b / 3a** show aggregate counts only. Hover shows counts.
- **Fig 2b** plots the 14 trials that report a numeric dose in a comparable unit
  (absolute particle counts or µg). Per-kg units assume a 70 kg adult.
  Trials reporting `mL`, `% PEP`, or `pmol phospholipid/kg` were omitted because
  they can't be put on the same axis without missing information.
- **Fig 3c** filters the source data to rows whose `Units` is `wt/wt` or `ug/ug`
  (true loading-capacity percentages). Rows in absolute units like g/EV or
  molecules/EV are excluded — they're per-particle amounts, not %LC.
- **Fig 3e** counts each paper once. The donut summarises the 98 unique siRNA
  studies; the table below lists every one and links to its source.
- **Fig 4 and 5 are restricted to the as-plotted subset.** The biodist sheets
  (Fig 4c, 4d, 5a, 5b, 5c) come from `Website_BioD_SD.xlsx`, which is the curated
  source data used to build the published figures (19–40 rows per sheet, matching
  the n's in the paper). Fig 4a and Fig 4b are taken from the bigger
  `Website_Source Data.xlsx`'s "Figure 4A,B" sheet but **filtered to studies whose
  DOI appears in the BioD_SD set**, so they reflect the same study population.
- **Fig 4a** dye-type labels are normalised (e.g. "Fluoroscent" → "Fluorescent dye",
  "Radio"/"radioactive" → "Radiolabel"). See `DYE_CANON` in `build_data.py` for the
  full mapping.
- **Fig 4b** plots only rows with a numeric dose. Weight-based and particle-based
  units are colour-coded; routes are bucketed (IV, IP, inhalation, etc.).
- **Fig 4c/d and Fig 5a/b** are biodistribution scatters. Source studies use many
  different signal units (photons/s, SNR, % ID/g, ...) so the y-axis is normalised
  within each study to the peak organ value. The raw value and its unit are in the
  hover tooltip so nothing is lost.
- **Fig 5c** uses the dedicated "Cancer Vs Non Cancer" sheet from BioD_SD (40
  studies, one % tumour value each) — matches the published Fig 5c view directly.
- **Fig 5d** is currently a placeholder pending the curated as-plotted list of
  engineered-vs-native studies.
